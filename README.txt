Инструкция по запуску:

1. Клонируем репозиторий
2. В редакторе кода (например vscode) переходим в папку проекта.
3. Запускаем Live Server
4. В браузере для обхода cors ставим плагин Cross Domain
5. Весь функционал рабочий по ТЗ.

При необходимости можно расширять и добавлять пограничные случаи и доп проверки, 
и играться с визуализацией.

Обнаружено ряд проблем в работе сервиса, при выполнении ТЗ:

1. После удаления всех задач и добавления новых, по API может подтянутся удаленная сделка с 1970 годом, 
хотя в интерфейсе ее уже нет, она была удалена.

2. После массового удаления и добавления сделок и задач, сервер начинает отдавать 204 код на последний эндпоинт
пришедший с сервера (количество сделок более 10). При уменьшении количесвта сделок работа восстанавливается.

3. В сервисе при просмотре всех сделок на низкой скорости сначала появляются стрелки вперед/назад,
затем исчезают. Однако сделок с самого начала мало, те стрелки не должны появляться на сколько я понимаю.
